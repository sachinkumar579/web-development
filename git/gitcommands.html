Git commands

  1. git init Creates a local git repository. Technically it creates a hidden .git folder
    • Create a folder - mkdir test-repo This is a windows command
    • Navigate into the directory cd test-repo This is a windows command
    • Make a git repo for tracking changes and uploading files to remote server git init This is a git command
  2. git clone This copies an existing Git repository. It has its own history ,manages its own files and is isolated environment from the original repository
    • git clone repo - cloning with repo url - downloads files from the specified repo url
    • git clone repo directory - clone with repo url to a specified directory
    • git clone --branch tag repo - clone a specific tag
    • git clone -depth=1 repo - shallow cloning - an example for cloning only with 1 commit id in history
    • git clone -b feature repo - clone specified branch from repo . -b option will fetch all branches and checkout the branch we choose
    • git clone --single-branch --branch branch repo - clone and fetch exclusively the branch we chose
  3. git add adds changes from working directory to the staging area
    • git add filename - adds filename to staging area
    • git add . - adds all files in working directory to staging area
  4. git stage Does the same thing as add. adds changes from working directory to the staging area. You can skip this command and go directly to git commit
  5. git status displays the state of the working directory and the staging area. see which changes have been staged, which haven’t, and which files aren’t being tracked by Git. Status output does not show you any information regarding the committed project history
  6. git commit Commits are snapshots of the project . We can recall commits or revert to older versions . The changes are not uploaded to remote repository yet
    • git commit -m "commit message" - takes all files in current staging area and creates a snapshot of changes .
  7. git push transfer commits from local repository to remote repository. Pushing exports commits from local branches to remote branches
    • creating remote branches
      • git remote add new-remote-repo url
      • git push new-remote-repo newbranchname - This command will push a copy of the local branch newbranchname to the remote repo new-remote-repo.
    • deleting remote branches git push origin --delete branchname
  8. git stash Temporarily stashes changes made to your working copy , It takes uncommitted changes , saves them for later use and reverts them from working copy
    • git stash
    • git stash pop
    • git stash apply
  9. git diff outputs changes between 2 data sets . we can give commit ids as input and it will show all changes
  10. git config Can be used for customizations
    • Selecting the editor for git
    • git aliases
  11. git fetch git fetch will retrieve the latest meta data info from original remote repo and it doesn't do any file transferring . Can be used to know the chages done in remote repo since the last pull
    • git fetch --prune Pruning during fetch is a cleanup method that deletes outdated remote references in your .git directory
  12. git pull git pull will bring the changes from remote repository . Files in working directory will get updated . Pull changes the file in current branch and working directory. it will download the remote content for the active local branch and immediately execute git merge to create a merge commit for the new remote content. If you have pending changes in progress this will cause conflicts and kick-off the merge conflict resolution flow
  13. git remote git remote command is used to create ,view and delete connections to other repositories . They can be used as reference for other remote repositories git remote -v - lists connections to other repositories with URLs
  14. git clean
    • git clean -n Removes local untracked files from current git branch . To see which files are deleted run git clean -n
    • git clean -f This command will delete local untracked files for real
  15. git branch this command lets you create , list , rename and delete branches
    • git branch - list all branches in the repository
    • git branch newbranch - create a new branch . doesn't checkout the new branch
    • git branch -d branchname - delete the specified local branch. Safe operation as it doesn't delete branches if it has unmerged changes (local commits)
    • git branch -D branchname - force delete even if it has unmerged changes (local commits)
    • git branch -m branchname - rename branch name
    • git branch -a - list all branches
    • git branch -r - list all remote branches
  16. git checkout branchname Switch branches
    • git checkout existingbranch
    • git checkout -b newbranch - The above example simultaneously creates and checks out new-branch. The -b option is a convenience flag that tells Git to run git branch before running git checkout new-branch.
    • git checkout commitid - this makes the working directory match the exact state of the commitid . To continue developing, you need to get back to the “current” state of your project- git checkout main
    • git checkout -t remotename/branchname switch to a remote branch. make sure to fetch remote branch with git fetch command first . t option stands for track. In modern versions of Git, you can then checkout the remote branch like a local branch.
    • git checkout -B branch commitid to checkout a new branch from a specific start point .
      To list commits done in the repository run git log --oneline --graph
    Make sure that the target branch exists by running git branch
  17. git cherry -v Check local commits
  18. git cherry-pick Apply selected commits from a branch
    • STEPS
    • git checkout develop - checkout to branch
    • git cherry-pick commitid - commitid - a commit id from some other branch
    • git push develop - to push local commit to remote develop branch
  19. git reset undo staging changes
    git reset --hard origin - removes local commits
  20. git merge to combine two branches
    • execute git checkout to switch to the receiving branch - git checkout main
    • Make sure the receiving branch and the merging branch are up-to-date with the latest remote changes. Execute git fetch to pull the latest remote commits. Once the fetch is completed ensure the main branch has the latest updates by executing git pull.
    • merge can be initiated by executing git merge branchname where branchname is the name of the branch that will be merged into the receiving branch (main branch). Resolve merge conflicts - add the changes to staging area and commit - git add . ; git commit -m "merging"
    • git push - to push commits to remote repository
  21. squash commits during merge
    • git checkout main - checkout main branch
    • git merge --squash develop - merge changes from develop to main
    • resolve merge conflicts if any and check staging status git status
    • git commit -m "squashed merges" - creates 1 commit with all changes from develop
    • git push main - push the changes from local to remote main branch
  22. git revert undo recent commit or a particular commit from the history- creates a new commit
    • git revert HEAD
    • git revert commitid
  23. git blame used to examine specific points of a file's history and get context as to who the last author was that modified the line git blame filename
  24. git log show commits for the currently selected branch
    • git log -n - show last n commits
    • git log --oneline- show commit ids with messages
    • git log --branches=*- view all commits across all branches by executing
    • git log -S To list all original commits in-which a specific code piece was added or modified execute git log with the -S option. Append the -S option with the code you are looking for git log -S "SearchText"
  25. git rebase transfer missing commits from one branch to another
    • git checkout feature - checkout branch to rebase
    • git rebase main - rebase commits from main branch to feature branch
    • if there are merge conflicts , resolve them and stage file. check status by running git status and don't commit yet
      • git rebase --continue - continue the rebase
      • git rebase --skip - skip the commit which caused merge conflict
    • You will see the rebase is completed with message "Successfully rebased and updated" . At this point feature branch in local is rebased . Don't pull the latest at this point. Run the command - git push --force-with-lease - run when there are no merge conflicts . push the local commits to remote feature branch. --force-with-lease - won’t overwrite any commits on the remote branch that someone else might have added while you were rebasing --force - will overwrite all commits in remote branch
  26. git tag Tags are ref's that point to specific points in Git history. Tagging is generally used to capture a point in history that is used for a marked version release (i.e. v1. 0.1). A tag is like a branch that doesn't change. Unlike branches, tags, after being created, have no further history of commits
    • git log --oneline - check if working directory is clean
    • git tag releaseV1 feature - this will tag the last commit on feature branch
    • git log --oneline - check if tag is added
    • git tag - to view all tags
    • git push --tags - to push tags to the remote repository
  27. git bisect walks you through recent commits ,asks you if they are good or bad and narrows down the broken commit
    • use git log to find a good commit and bad commit
    • Assuming newest commit is bad and the first commit is good
    • git bisect start
    • git bisect good commitid - let the bisect know of a good commit
    • git bisect bad commitid - let the bisect know of a bad commit. This will checkout an old commit for us - the commit halfway between good commit and bad commit - it cuts the commit history down in halves untill it finds the bad commit
    • If this commit is bad run git bisect bad or if it is good run git bisect good
    • Repeat untill bisect finds the bad commit . It prints the first bad commit - commitid is the first bad commit
    • To end run git bisect reset . It will checkout the current branch
    • To know more about the bad commit run git show commitid
